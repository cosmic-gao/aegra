## Status definitions

| Status | Meaning |
|:--|:--|
| **Supported** | Available in the current release and tested. |
| **Coming soon** | Actively being worked on or has a concrete plan. |
| **Not yet planned** | Not on the immediate roadmap. May be added based on demand. |

## Feature matrix

### Assistants and graphs

| Feature | Status | Notes |
|:--|:--|:--|
| Assistants CRUD | Supported | Create, read, update, delete, and search assistants. |
| Assistant versioning | Supported | Each update creates a new version. Pin or roll back. |
| Graph schemas | Supported | Input, output, state, and config schemas exposed via API. |
| Graph visualization | Supported | Mermaid graph structure returned by the API. |

### Threads and state

| Feature | Status | Notes |
|:--|:--|:--|
| Thread CRUD | Supported | Create, read, update, delete, and search threads. |
| State persistence | Supported | PostgreSQL checkpoints via LangGraph. |
| Thread state inspection | Supported | Get current state, values, and next nodes for any thread. |
| Thread history | Supported | Full checkpoint history with replay capability. |
| Thread copy | Supported | `POST /threads/{id}/copy` forks a thread with its full checkpoint history into a new thread. |
| Thread prune | Supported | `POST /threads/prune` deletes threads by id, or `keep_latest` drops all but the newest checkpoint per namespace. |
| Thread search projection | Supported | `select` projects fields, `extract` pulls dot/bracket paths, and `values`/`ids` filter on the latest state. Latest state is materialized into a 1:1 `thread_state` table (not the wide thread row), so list/search omit `values`/`interrupts` unless `select`ed; the checkpointer stays the source of truth. |
| Per-thread TTL | Supported | `ttl` on create/update sets retention; `include=ttl` returns it. |
| Thread supersteps | Supported | `supersteps` on create seeds the thread's state (Studio initial state). |

### Runs and execution

| Feature | Status | Notes |
|:--|:--|:--|
| Background runs | Supported | Non-blocking execution with polling. |
| Streaming (7 SSE modes) | Supported | values, updates, messages, messages-tuple, custom, events, and debug. |
| SSE reconnection | Supported | Resume interrupted streams from where they left off. |
| Sync wait | Supported | Block until run completes and return final output. |
| Subgraph streaming | Supported | Stream events from nested subgraphs. |
| Stateless runs | Supported | Execute without persisting thread state. |
| Cron / scheduled jobs | Supported | Trigger runs on a schedule. Standard and seconds-level cron expressions. IANA timezone support. |
| Double-texting | Supported | `multitask_strategy`: `reject` (default) / `interrupt` / `rollback` / `enqueue`. At most one run executes per thread at a time. |
| Cancel | Supported | `POST .../cancel` with `action`: `interrupt` / `cancel` / `rollback` (rollback deletes the run and its checkpoints). |
| Bulk cancel | Supported | `POST /runs/cancel` cancels by `status` (pending/running/all) or `thread_id` + `run_ids`. |
| Batch runs | Supported | `POST /runs/batch` creates many stateless background runs in one request (bounded concurrency). |
| Delayed runs | Supported | `after_seconds` schedules a run to start after a delay; a background scheduler submits it when due. |
| Checkpoint durability | Supported | Per-run `durability`: `async` (default) / `sync` / `exit`, forwarded to the LangGraph runtime. `checkpoint_during` is accepted as a deprecated alias. |
| Webhook callbacks | Supported | POST the final Run payload to a `webhook` URL on terminal state. Bounded retry with jitter (tenacity), optional HMAC signing, SSRF guard. |
| `if_not_exists` | Supported | Run creation on a missing thread: `reject` (default, 404) or `create`. |
| Thread count | Supported | `POST /threads/count` returns the number of threads matching the search filters. |

### Scaling and reliability

| Feature | Status | Notes |
|:--|:--|:--|
| Worker job queue | Supported | Redis BLPOP with configurable worker count and concurrency. |
| Concurrent execution | Supported | Up to `N_JOBS_PER_WORKER` (default 10) concurrent runs per worker. |
| Lease-based crash recovery | Supported | Automatic re-execution of runs from crashed workers via checkpoint resumption. |
| Cross-instance cancel | Supported | Cancel propagates via Redis pub/sub to any instance. |
| Job timeout | Supported | Configurable max execution time (default 1 hour). |
| Postgres fallback | Supported | Workers fall back to DB polling when Redis is unavailable. |
| Multi-instance SSE | Supported | SSE streams work across instances via Redis broker. |

### Human-in-the-loop

| Feature | Status | Notes |
|:--|:--|:--|
| Interrupt before / after | Supported | Pause execution at specific nodes. |
| User approval gates | Supported | Resume or reject interrupted runs. |
| State editing | Supported | Modify thread state before resuming. |

### Storage

| Feature | Status | Notes |
|:--|:--|:--|
| Key-value store | Supported | Namespaced key-value storage with listing and search. |
| Semantic search | Supported | Vector embeddings via pgvector. Configurable embedding models. |
| Store TTL | Supported | `store.ttl` config plus per-request `ttl`/`index` on put and `refresh_ttl` on get/search. |
| Thread retention (TTL) | Supported | Opt-in `CHECKPOINTER_TTL_ENABLED`: a sweeper deletes stale threads with no active run and their checkpoints. |
| Thread state materialization | Supported | Latest state materialized into `thread_state` (1:1) for search, with a values-hash skip on unchanged writes. `THREAD_STATE_MATERIALIZE=false` opts into pure checkpointer-centric (GET reads state from the checkpointer; `values` search unsupported). |
| Encryption at rest | Supported | Set `LANGGRAPH_AES_KEY` to encrypt checkpoint state via langgraph's `EncryptedSerializer` (AES). |

### Authentication and authorization

| Feature | Status | Notes |
|:--|:--|:--|
| Configurable auth handlers | Supported | Write Python functions for JWT, OAuth, Firebase, or custom schemes. |
| Fine-grained authorization | Supported | Per-resource access control based on user identity. |
| Studio auth bypass | Supported | Optional flag to disable auth for local development. |

### Extensibility

| Feature | Status | Notes |
|:--|:--|:--|
| Custom routes | Supported | Add FastAPI endpoints alongside the Agent Protocol API. |
| Shared dependencies | Supported | Load Python modules available to all graphs. |
| MCP server support | Supported | `/mcp` Streamable HTTP endpoint exposes **each graph as its own tool** (tool name = graph id, input schema = the graph's input schema), matching the LangGraph Platform MCP model. Stateless per call, authenticated with the same auth backend as the REST API (tools run under the caller's identity). Toggle with `MCP_ENABLED` or `http.disable_mcp`. |
| A2A protocol | Supported | `/a2a/{assistant_id}` exposes each assistant as an A2A agent (agent cards under `/.well-known`), built on the official `a2a-sdk` and authenticated with the same backend as REST. Toggle with `A2A_ENABLED`. |
| Configurable headers | Supported | `http.configurable_headers` (includes/excludes) forwards allowlisted request headers into each run's `config.configurable`. |
| WebSocket streaming | Supported | `/threads/{id}/stream/events` also serves the Agent Protocol v2 WebSocket transport used by the latest SDKs. |

### Not yet planned

| Feature | Status | Notes |
|:--|:--|:--|
| RemoteGraph | Not yet planned | Call remote graphs as local subgraphs. |
| Rate limiting | Not yet planned | Per-user or per-key request throttling. |
| Multi-org workspaces | Not yet planned | Tenant isolation within a single deployment. |
